Kako LLM-ovi rade (kratka verzija)

  • LLM je istreniran na ogromnoj količini teksta i uči da predvidi sledeći token.
  • Nema memoriju između zahteva - svaki poziv je stateless.
  • Context window je sav tekst koji model može da "vidi" odjednom - istorija, uputstva i odgovor zajedno.
  • System prompt definiše ličnost, pravila i ulogu pre nego što razgovor počne.
  • Temperature kontroliše nasumičnost - niska = precizno, visoka = kreativno.
Model sam po sebi ne zna ko si, na kom projektu radiš, niti kako želiš da dobijаš odgovore - osim ako mu ne kažeš.

Zašto je ovo važno za wrapper

Sve što model "zna" o tvom kontekstu mora biti poslato uz zahtev. Zadatak wrappera je da to upravlja automatski - da se ne moraš ponavljati svaki put.

Šta je wrapper?

ChatGPT u browseru

  • Kucaš, odgovara
  • Nema memorije između sesija
  • Ne može da vidi tvoje fajlove ili kod
  • Fiksna ličnost, fiksan UI
  • Bez automatizacije - treba čovek
  • Ne može se povezati sa drugim alatima

Tvoj sopstveni wrapper

  • Ti kontrolišeš sve oko modela
  • Trajna istorija razgovora po sesiji
  • Može da učita i razmišlja o pravim fajlovima
  • Prilagođena ličnost ugrađena trajno
  • Može da radi automatski, bez kucanja
  • Može se ulančati sa skriptama i pipeline-ovima
Wrapper ne menja model - menja sve oko modela. Tu leži prava moć.

Kako je BorisBot napravljen

class BorisBot: def __init__(self, api_key): self.client = OpenAI( api_key=api_key, base_url="https://api.groq.com/openai/v1" ) self.history = [system_prompt] # memorija self.loaded_files = [] # učitani fajlovi def chat(self, message): self.history.append(user_msg) reply = self.client.chat.create( model="llama-3.1-8b-instant", messages=self.history # cela istorija svaki put ) self.history.append(reply) return reply

Model

Llama 3.1 8B na Groq-u - besplatno, brzo, isti API kao OpenAI. Jedna linija koda za zamenu modela.

Memorija

self.history raste sa svakim redom. Ceo razgovor se šalje uz svaki zahtev - tako funkcioniše chat kroz više krugova.

Ličnost

System prompt se ubacuje jednom pri pokretanju. Model ostaje u ulozi tokom cele sesije.

BorisBot komande

/load <putanja> Učitava fajl sa diska u kontekst razgovora. Zatim postavljaš pitanja o njemu prirodno.
/loaded Prikazuje koji fajlovi su trenutno učitani u sesiji.
/atlas Ispisuje lični Atlas & Unity vodič za komande - rebuild, dev server, testovi.
/joke Traži od bota vic.
/model Prikazuje koji model radi i gde je hostovan.
/clear Resetuje istoriju razgovora. Svež start bez ponovnog pokretanja procesa.

Zašto prilagođene komande?

Svaka komanda je samo Python - provjera stringa koja transformiše unos pre nego što stigne do API-ja, ili ga zaobilazi potpuno za lokalne podatke kao /atlas.

Dodavanje nove komande je 5 linija koda. Bez pluginova, bez marketplace-a, bez dozvola.

Ti poseduješ interfejs. ChatGPT nema /atlas komandu. Tvoj ima.

Komanda /load - zašto je važna

  • Čita bilo koji fajl sa lokalnog diska - Python, SQL, config, logovi, šta god.
  • Ubacuje ceo sadržaj direktno u istoriju razgovora.
  • Model sada ima taj fajl u svom context window-u - može da razmišlja o njemu, objasni ga, pronađe bugove.
  • Ostaje dok ne uradiš /clear - nema ponovnog učitavanja.
# Primer sesije /load ~/projects/atlas/models.py → Učitan models.py (312 linija) Ti: kako izgleda User model? Ti: objasni vezu između User i Team modela

Ovo je prava razlika

ChatGPT ne može da otvori tvoje fajlove. Ne može da vidi tvoj kod. Moraš ručno da lepiš kod - i zaboravlja između sesija.

Sa /load, usmeravaš bota na pravi kod i vodiš pravi razgovor o njemu.

Isti model. Potpuno drugačija mogućnost - jer ti kontrolišeš kontekst.

Šta ovo znači u praksi?

14.400 zahteva dnevno = praktično neograničeno za ličnu upotrebu. Prosečan razgovor ima 10–30 poruka.

500k tokena dnevno

Jedan token ≈ 0.75 reči. 500k tokena = ~375.000 reči. To je oko 750 stranica teksta - na dan, besplatno.

Nema kreditne kartice. Nema troška. Samo se registruješ na console.groq.com i dobijaš API ključ.

Zašto uopšte praviti wrapper?

Vlasništvo nad kontekstom

Tvoj stack, tvoji projekti, tvoj stil - ugrađeni trajno. Nikad se ne ponavljaš. Svaka sesija počinje sa punim kontekstom o tome ko si i šta praviš.

Automatizacija

Wrapper je samo Python. Možeš ga pozvati iz skripte, git hook-a, cron joba. Model radi bez čoveka koji kuca - pokreni ga po rasporedu, prosledi rezultate na Slack, okini na događaje.

Sloboda izbora modela

Promeni sa Llama na GPT-4 na Gemini u jednoj liniji. Bez ponovnog učenja UI-ja. Bez migracije podešavanja. Model je samo parametar - tvoja logika, komande i format istorije ostaju isti.

Wrapper pretvara generički model u specijaliste koji poznaje tvoj kod, govori tvojim jezikom i radi unutar tvojih alata.

Šta smo napravili i šta sledi

Šta BorisBot ima danas

  • Besplatan API preko Groq-a - Llama 3.1, nula troška
  • Trajna memorija razgovora po sesiji
  • Prilagođena ličnost i system prompt
  • Učitavanje fajlova - razmišljanje o pravom kodu
  • Brojač krugova, obojen ispis, čist CLI
  • Lične komande: /atlas, /load, /joke
  • Konfiguracija preko .env - bez hardkodovanih tajni

Laki sledeći koraci

Automatsko učitavanje fajlova projekta pri pokretanju. Čuvanje razgovora u log fajl. Dodaj /git komandu koja pokreće git status i objašnjava ga.

Veći sledeći koraci

Daj mu prave alate - zapravo pokreni terminalne komande, pozovi API-je, piši fajlove. To je kad chatbot postaje agent.
